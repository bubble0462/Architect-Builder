# /executetask — 任务执行

当用户调用此技能时，执行以下流程：

## 步骤 1：读取任务文件

必须读取以下文件：

1. `.ai/PLAN.md` — 了解整体方案
2. `.ai/TASK.md` — 了解具体任务
3. `.ai/ACCEPTANCE.md` — 了解验收标准

## 步骤 2：确认理解

向用户复述你的理解，确保一致：

```markdown
我理解的任务是：
1. [任务1概要]
2. [任务2概要]

影响的文件：
- [文件列表]

我会严格按照 TASK.md 执行，遇到不确定的地方会问你。
请确认是否可以开始？
```

## 步骤 3：创建 git 分支

```bash
git checkout -b feat/<task-name>
```

## 步骤 4：执行任务

### 执行规则

**必须做：**
1. 严格按 TASK.md 修改代码
2. 不修改无关文件
3. 遇到不确定的地方，问用户
4. 每完成一项，更新 IMPLEMENTATION.md
5. 完成后运行测试或构建
6. 生成 diff 文件供审查

**禁止做：**
1. 不重新规划任务
2. 不修改 TASK.md
3. 不添加任务外的功能
4. 不跳过测试
5. 不猜测用户意图 — 不确定就问

### 遇到问题时

如果遇到以下情况，**必须问用户**：

1. **需求不明确** — 不确定要实现什么
2. **技术方案有歧义** — 有多种实现方式
3. **发现潜在问题** — 可能影响其他功能
4. **缺少必要信息** — 需要更多上下文
5. **超出能力范围** — 需要特殊权限或工具

**提问格式：**
```markdown
我在执行 Task X 时遇到以下问题：

1. [问题描述]
   - 选项 A: [说明]
   - 选项 B: [说明]
   
请问你希望怎么做？
```

## 步骤 5：更新实现记录

### .ai/IMPLEMENTATION.md

```markdown
# 实现记录

## 完成的任务
- [x] Task 1: [名称] — [简要说明]
- [x] Task 2: [名称] — [简要说明]

## 修改的文件
- `src/xxx.py`: [修改内容]
- `src/yyy.py`: [修改内容]

## 测试结果
- [ ] 单元测试: [通过/失败]
- [ ] 集成测试: [通过/失败]

## 遇到的问题
- [问题描述及解决方式]

## 备注
- [其他需要说明的]
```

## 步骤 6：生成 diff 文件

```bash
git diff main > .ai/IMPLEMENTATION.diff
```

如果是在特性分支上：
```bash
git diff main...HEAD > .ai/IMPLEMENTATION.diff
```

## 步骤 7：提交代码

```bash
git add .
git commit -m "feat: implement [task name]"
```

## 步骤 8：告知用户

完成后向用户汇报：
1. 完成了哪些任务
2. 修改了哪些文件
3. 测试结果
4. diff 文件位置
5. 是否需要审查

---

## 注意事项

- **不要猜测** — 不确定就问用户
- **保持专注** — 只做 TASK.md 里的任务
- **留痕** — 所有修改都要有记录
- **可回滚** — 用 git 分支，方便回退
